# Первый запуск

## Splash-экран

При старте лаунчер показывает splash-экран, пока проверяет наличие сохранённого токена. Если токен есть и валиден — переходит на Home. Если нет — на Auth.

## Авторизация

Вводишь адрес сервера (если не задан в конфиге), логин и пароль. Лаунчер отправляет запрос через текущий адаптер и получает `AuthToken`.

Токен сохраняется в `FlutterSecureStorage` (на Windows — через DPAPI). При следующем запуске авторизация не нужна.

## Главный экран

После логина видишь список серверов с онлайном. Кликаешь на сервер — открывается его страница с профилями.

## Синхронизация и запуск

<Steps>
<Step>
Выбираешь профиль на странице сервера
</Step>
<Step>
Нажимаешь «Играть» — запускается `syncAssets`, скачиваются недостающие файлы, проверяются хеши
</Step>
<Step>
Pre-launch проверка целостности (LaunchGuard) — если файлы повреждены, запуск блокируется
</Step>
<Step>
Формируются аргументы запуска (JVM args, classpath, game args) и стартует Java-процесс
</Step>
<Step>
Включается мониторинг: ProcessGuard следит за DLL-инжектом, RuntimeWatcher — за файлами. Консольный вывод доступен на экране Console.
</Step>
</Steps>
