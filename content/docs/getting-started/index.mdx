# Noctis

Noctis — десктопный лаунчер для Minecraft. Подключается к серверам через адаптеры (Gravit, REST API или мок для разработки), проверяет целостность файлов, скачивает обновления и запускает игру.

## Что умеет

<Cards>
<Card title="Мультибэкенд" icon={<Gamepad2 />}>
Работает с GravitLauncher (WebSocket), кастомным REST API или моком. Переключается одной настройкой.
</Card>
<Card title="Автообновление" icon={<RefreshCw />}>
Синхронизация файлов с сервера, SHA-1 проверка после скачивания.
</Card>
<Card title="Система тем" icon={<Palette />} href="/docs/themes">
UI полностью заменяемый. Из коробки — Void Default с glassmorphism.
</Card>
<Card title="Защита от читов" icon={<Shield />} href="/docs/security">
Pre-launch проверка файлов, детект DLL-инжекта, мониторинг файлов во время игры.
</Card>
<Card title="Мультиплатформенность" icon={<Monitor />}>
Windows (основная), Linux, macOS.
</Card>
</Cards>

<Callout title="Для разработчиков">
Если ты хочешь разрабатывать под Noctis — начни с раздела [API](/docs/api), а для кастомизации UI — с [Тем](/docs/themes).
</Callout>

## Стек

| Технология | Для чего |
|-----------|----------|
| Flutter / Dart | UI и логика |
| Riverpod | Стейт-менеджмент и DI |
| GoRouter | Навигация |
| Freezed | Иммутабельные модели |
| Dio | HTTP-клиент |
| dart:ffi | Нативные вызовы Windows API |

## Архитектура

Noctis построен на Clean Architecture:

```
domain/     — бизнес-логика, сущности, интерфейсы репозиториев
data/       — реализация репозиториев, работа с хранилищем
core/       — адаптеры, DI, конфиг, ошибки, security, темы
```

Зависимости идут только внутрь: `presentation → domain ← data`. Domain не знает ни про Flutter, ни про сеть.
