# ProcessGuard

Мониторит загруженные модули (DLL) игрового процесса через Windows API. При обнаружении нового модуля после стабилизации — убивает процесс.

## Windows API (FFI)

Используемые функции из `kernel32.dll` и `psapi.dll`:

| Функция | Назначение |
|---------|-----------|
| `OpenProcess` | Получить хэндл процесса |
| `EnumProcessModulesEx` | Список загруженных модулей |
| `GetModuleFileNameExW` | Путь к модулю |
| `CheckRemoteDebuggerPresent` | Проверка дебаггера |
| `TerminateProcess` | Убить процесс |

## Фазы работы

### 1. Baseline (сбор модулей)

После запуска Java-процесса ProcessGuard каждые 2 секунды снимает snapshot загруженных модулей. JVM динамически подгружает библиотеки при старте (jvm.dll, lwjgl, OpenAL и т.д.).

Когда количество модулей стабилизируется (3 одинаковых подряд snapshot-а) — baseline заморожен.

### 2. Active (мониторинг)

Каждые 2 секунды:
- Получает текущий список модулей
- Сравнивает с baseline
- Проверяет `CheckRemoteDebuggerPresent`

### 3. Violation → Kill

Любой новый модуль, не входящий в baseline = `DllInjection` violation → `TerminateProcess`.
Обнаружен дебаггер = `DebuggerAttached` violation → `TerminateProcess`.

## Ограничения

- Manual mapping (без LoadLibrary) не детектится — модуль не появляется в списке
- Kernel-level читы работают в ring-0, мы в ring-3
- Покрывает ~90% распространённых инжекторов для Minecraft
